Installation

First Steps

I added an “Alias” that needs to be run on first logon to fix a bug with GRUB - just run “grubfix” command in Terminal then reboot. There are tons of other aliases - type “alias” in Terminal to see them all.

Very Important : Make sure to read each section’s description, and note if you end up stuck with Desktop not loading, below under nVidia section you might get some clarification. Again, READ ENTIRE PAGE else I cannot help, and Google is your best friend...

During installation you will have the option to install many packages, which will be pulled from the Internet, so you will have to have an active Internet connection otherwise nothing will be installed.

Package List Explained

The list is Dynamic so it will be constantly changing, meaning that if you install it today, and I add/change a few things installing it in the future will present you with a different list, much the same but with a few packages removed or added, I did it this way so I don’t have to re-compile and re-upload ISO as I do not have enough bandwidth to spare, but for now I will try to explain below some of the sections below :

- Core : That’e where you select your Kernel & drivers. If you want latest Stock Kernel, or any other one check one you prefer. AMD/Intel/nVidia ? Select one you got. Just don’t skip nVidia part below before going ahead to next section. Also know that you can 2 Kernels installed at the same time you will be prompther during Boot which to boot from ;)

Note : Don’t forget to select your Processor’s Microcode, it’s very important !

- Apps : Here you can choose what apps you want out of the box. Many to select from. All very useful. I don’t put useless stuff.

- Gaming : If you want games, here’s where you select Launcher of choice with some “Native” games. Note if you select games it will make installation take much longer.

- Extras : This is where you can select a few more stuff I consider Extras. Like some custom tools by ArcoLinux and some themes.

Dynamic List can also be benefical in such a way that you can request for a certain package to be added to the list, after I have tested and made sure it’s useful and is bug free of course, at which point I will update list with it included, then on your end you don’t have to re-download updated ISO, saving us both the hassle, so next time you install my Distro, package(s) will be present for you to select during installation. Win-Win ;)

No Internet Connection

Some users out there have reported an issue where Calamares installer was showing a message saying it’s wasn’t able to establish a connection to the Internet. In that case you can still go ahead with installation, only caveat with this is, you cannot install any of the packages in the list. So go ahead with installation, do not check any of the boxes as it will not work spitting an error.

These are the steps, quit installer, then connect to net n hit F12 type in the command below, when Installer launches see if it still says no Internet. Or, reboot from ISO immediately quit installer n do as I said earlier.

Command

calamares_polkit -d

Not in my hands developer of installer hasn’t fixed issue yet but he said it’s coming. So we have to wait for that. Otherwise I have no idea why it happens or how to fix it beyond that… Will have to wait and see, till then install without selecting anything from list :(

nVidia GPUs

Now it’s a known fact that nVidia don’t play well with Linux, so if you have an old Laptop/Desktop with an older GPU or even Hybrid Intel/nVidia requiring 390x Drivers, and you get stuck on Black Screen Of Death, then you will need to Google your way out of it as I do not have enough expertise on the subject.

All I know is that you might need to try installing it without nVidia drivers, or with just the Intel ones. It’s a Trial & Error sort of deal. Just an FYI, selecting Microcode+Kernel without selecting any GPU Drivers especially on Hybrid setups could be the safest route, GPU drivers are there for those who need them, like 10 Series nVidia and newer... My Distro uses SDDM not LightDM, for compatibility. Best of luck on your journey ;)
